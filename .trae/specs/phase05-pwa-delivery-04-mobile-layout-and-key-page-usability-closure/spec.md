# Phase05 PWA 移动端布局与关键页面可用性收口 Spec

## Why

`phase05-pwa-delivery-03` 已完成安装壳、`service worker`、更新策略与最小离线兜底闭环，但当前仓库的移动端体验仍存在“能安装、能打开，不一定稳定好用”的现实风险。进入 `phase05-04` 需要把关键业务页面的真实手机可用性、布局切换、`safe-area`、软键盘遮挡与视口行为收口成最小、统一、可解释的移动端主线，避免安装态和浏览器态继续出现明显行为分叉。

## What Changes

- 审计并收口 `AppLayout`、`MobileLayout`、`UnifiedNavigation`、`globals.css` 与关键页面的移动端行为
- 冻结关键页面在主流手机尺寸下的最小可用性标准，优先覆盖首页 / Dashboard、列表、表单、详情、搜索与提醒相关页
- 允许减少客户端判屏导致的闪烁、布局跳变和安装态/浏览器态不一致
- 允许补最小视口、`safe-area`、软键盘遮挡与底部导航避让优化
- 允许做不改变既有视觉语言的最小信息架构与交互调整
- **BREAKING**：移动端页面布局行为必须统一到单一 Web UI 主线，不再允许“安装态一套临时补丁、浏览器态另一套行为”的隐性双实现继续演化

## Impact

- Affected specs:
  - 移动端布局主线
  - `safe-area` 与视口行为
  - 软键盘遮挡与底部导航避让
  - 首页 / Dashboard 移动端可用性
  - 列表页、表单页、详情页、搜索页与提醒相关页移动端可用性
- Affected code:
  - `src/components/layout/AppLayout.tsx`
  - `src/components/layout/MobileLayout.tsx`
  - `src/components/layout/UnifiedNavigation.tsx`
  - `src/app/globals.css`
  - 必要时关键业务页面、列表组件、表单组件与详情组件

## ADDED Requirements

### Requirement: 移动端布局主线统一

系统 SHALL 在安装态与浏览器态下复用同一套业务 UI 主线，并将布局差异收敛到最小必要范围。

#### Scenario: 安装态与浏览器态布局一致

- **WHEN** 用户在同一台手机上分别以安装态和浏览器态访问系统
- **THEN** 页面整体信息结构、主操作入口与底部导航行为保持一致
- **AND** 不会因为安装态切换而进入另一套布局体系或另一套业务页面表达

### Requirement: 关键页面具备可接受移动端可用性

系统 SHALL 使关键业务页面在主流手机尺寸下具备可接受可用性，优先覆盖首页 / Dashboard、列表、表单、详情、搜索与提醒相关页。

#### Scenario: 关键页面在手机尺寸下可用

- **WHEN** 用户在主流手机尺寸下访问关键页面
- **THEN** 页面无严重横向溢出
- **AND** 核心信息与主要操作入口在首屏或合理滚动范围内可达
- **AND** 不会因为布局切换导致主要按钮、筛选、输入项或关键信息不可用

### Requirement: Safe Area、底部导航与软键盘遮挡受控

系统 SHALL 控制 `safe-area`、底部导航与软键盘对内容区的挤压和遮挡，避免移动端主要操作被覆盖。

#### Scenario: 输入与操作不被遮挡

- **WHEN** 用户在移动端进行表单录入、搜索或编辑操作并唤起软键盘
- **THEN** 当前输入区域与提交操作仍可访问
- **AND** 底部导航与安全区不会长期遮挡主要操作
- **AND** 页面不会因键盘弹出而产生明显不可恢复的布局跳变

### Requirement: 客户端判屏闪烁与不一致最小化

系统 SHALL 尽量减少由客户端判屏、安装态识别或视口变化导致的闪烁、不一致和跳变。

#### Scenario: 页面初始渲染稳定

- **WHEN** 用户首次打开应用、刷新页面或在安装态与浏览器态之间切换
- **THEN** 布局切换不会产生明显闪烁或大范围重排
- **AND** 关键内容不会在初始渲染后发生令人困惑的位置跳动

## MODIFIED Requirements

### Requirement: 移动端可用性从“基础适配存在”升级为“关键页面可稳定使用”

项目的移动端适配 SHALL 从“已有响应式布局、底部导航与基础表单优化”升级为“关键页面在真实手机场景下可以稳定使用”，并与 `phase05` 的安装交付主线保持一致。

#### Scenario: PWA 不只是能安装

- **WHEN** 用户已经完成安装并在手机上进入真实业务流
- **THEN** 应用不仅能打开，还能完成查询、列表浏览、详情查看、搜索、提醒查看和基础录入
- **AND** 用户不会因为布局缺陷而必须退回桌面浏览器才能完成核心操作

## REMOVED Requirements

### Requirement: 只要页面具备基础响应式能力即可视为移动端可用

**Reason**: 当前阶段已经明确，PWA 手机端交付不能只停留在“页面可以缩放打开”，而必须证明关键页面在真实手机场景下可稳定使用。

**Migration**: 将移动端可用性的真相源统一收口到 `AppLayout`、移动导航、`globals.css` 与关键页面组件；后续仅在这一统一布局主线中做最小优化，不再通过零散页面补丁维持表面可用。
