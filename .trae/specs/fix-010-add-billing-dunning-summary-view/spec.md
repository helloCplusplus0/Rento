# 本次应催缴账单汇总视图 Spec

## Why
当前 Rento 仅供管理者本人使用，租客没有自助入口。管理者需要在合同详情页中快速生成一页适合截图发送给租客的“本次应催缴账单汇总”，以降低线下微信催缴时的重复操作成本，同时不冲击现有独立账单事实模型。

## What Changes
- 在合同详情页新增“本次应催缴账单汇总”入口
- 基于同一合同下的原始独立账单生成只读汇总视图
- 新增“默认按催缴时间窗口预选 + 管理者手动调整”的选择流程
- 复用当前水电聚合账单的展示模板或高度一致的共享 UI 模板
- 明确区分“只读催缴汇总视图”和“正式账单详情”
- **BREAKING** 无

## Impact
- Affected specs:
  - 合同详情页操作入口
  - 账单选择与本次应催缴默认筛选规则
  - 汇总视图展示模板复用
  - 账单语义边界说明
- Affected code:
  - `src/components/business/EnhancedContractDetail.tsx`
  - `src/components/business/BillBasicInfo.tsx`
  - 相关汇总数据转换逻辑与辅助组件
  - 如有必要，新增仅服务于只读汇总页的轻量前端状态组件

## ADDED Requirements
### Requirement: 合同详情页提供本次应催缴账单汇总入口
系统 SHALL 在合同详情页提供进入“本次应催缴账单汇总”流程的入口，用于生成适合截图发送的只读汇总页。

#### Scenario: 进入汇总流程
- **WHEN** 管理者进入某个合同详情页并打开“本次应催缴账单汇总”
- **THEN** 系统展示该合同下可参与本次催缴汇总的候选账单列表
- **AND** 不修改任何原始账单状态

### Requirement: 默认预选遵循本次催缴时间窗口
系统 SHALL 根据可解释的“本次催缴时间窗口”默认预选账单，而不是默认汇总该合同下的全部待处理账单。

#### Scenario: 分期租金场景下避免误选未来期账单
- **WHEN** 合同存在多期已预生成但尚未到当前催缴时点的租金账单
- **THEN** 系统默认只预选落在本次催缴时间窗口内的账单
- **AND** 不把未来期账单自动并入本次汇总

#### Scenario: 默认预选结果可解释
- **WHEN** 管理者查看默认预选结果
- **THEN** 该结果必须能被解释为“同一合同下当前应催缴的一组原始账单”

### Requirement: 管理者可在默认预选基础上自由调整
系统 SHALL 允许管理者在默认预选基础上手动勾选或取消账单，以适配实际催缴场景。

#### Scenario: 管理者调整本次催缴账单集合
- **WHEN** 管理者勾选或取消某张候选账单
- **THEN** 汇总结果实时更新
- **AND** 最终汇总页只反映本次实际选中的原始账单集合

### Requirement: 汇总页复用水电聚合账单展示模板
系统 SHALL 复用当前水电聚合账单的展示模板或保持高度一致的共享模板，以减少额外 UI 设计偏差。

#### Scenario: 汇总页视觉结构保持一致
- **WHEN** 管理者打开本次应催缴账单汇总页
- **THEN** 页面应呈现与当前水电聚合账单一致或高度一致的摘要、金额和分项结构
- **AND** 页面可直接用于手机截图发送

### Requirement: 汇总页是只读视图而不是新账单事实
系统 SHALL 明确把本次应催缴账单汇总定义为只读展示视图，而不是新的账单记录或正式账单详情页。

#### Scenario: 退出汇总页后原始账单事实不变
- **WHEN** 管理者关闭或离开汇总页
- **THEN** 原始 `Bill` 记录、账单历史、账单状态和统计主链保持不变

## MODIFIED Requirements
### Requirement: 合同详情页账单相关操作的边界说明
合同详情页中的新增汇总入口只服务于管理者的本次催缴展示需求，不应改变原有账单历史、账单详情跳转与账单统计语义。

#### Scenario: 账单历史保持事实视图
- **WHEN** 管理者查看账单历史 tab
- **THEN** 仍然只看到原始账单事实列表
- **AND** 不会把临时催缴汇总结果写入账单历史

## REMOVED Requirements
### Requirement: 默认汇总全部待处理账单
**Reason**: “全部待处理账单”会误伤分期租金和未来期账单，无法正确表达“本次应催缴”。
**Migration**: 改为“按催缴时间窗口默认预选 + 管理者自由调整”的汇总模式。
