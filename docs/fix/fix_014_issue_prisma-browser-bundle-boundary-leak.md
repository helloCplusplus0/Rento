# Fix 014 Issue - 前端浏览器包误引入 Prisma 导致线上首页白屏

## 1. 问题摘要
- 问题编号：`fix_014`
- 发现阶段：真实场景验证
- 问题级别：`P0`
- 发现日期：2026-06-09
- 发现人：honlink

## 2. 背景
- 业务背景：
  - 当前 `Rento-miniX` 已在真实云服务器上按 GitHub Release 部署包完成正式部署验证，目标是以 `Caddy + systemd + Hono + PostgreSQL` 作为正式部署主线。
  - 当前阶段默认工作流已进入 `phase16-parity-verification-cutover-and-legacy-exit`，真实部署可用性是是否能够进一步进入 cutover 复判的核心前提之一。
  - 本轮线上反馈显示：部署后访问根路径 `https://rento2026.top/` 出现白屏，而不是单一功能页局部异常，说明问题已触及前端运行时主入口。
- 涉及模块：
  - `src/minix` 前端运行时入口
  - React Router 路由装载
  - 账务兼容语义层
  - 退租结算共享逻辑
  - Prisma 相关共享类型与服务端领域模块边界
- 涉及页面 / API：
  - `/`
  - `/bills`
  - `/contracts/:id/checkout`
  - 浏览器前端首屏启动链路
- 涉及样例数据：
  - 已部署到 `https://rento2026.top/` 的真实云服务器正式环境
  - 已登录或未登录时均会加载前端入口资源的正式浏览器访问链路

## 3. 复现前提
- 环境：真实云服务器正式部署环境
- 账号：无需特定账号，访问根路径即可触发前端入口加载
- 前置数据：
  - 已完成 Release 部署包拉取与服务启动
  - 域名与 HTTPS 访问正常连通
  - 浏览器可成功请求首页 HTML 与前端静态资源
- 是否稳定复现：是

## 4. 复现步骤
1. 在真实云服务器完成当前 `Rento-miniX` Release 部署包部署，并确保站点可通过 `https://rento2026.top/` 访问。
2. 使用浏览器打开 `https://rento2026.top/`。
3. 观察页面表现与浏览器控制台输出。

## 5. 预期结果
- 访问根路径时，前端应用应正常启动并进入登录页或工作台，而不应在浏览器初始化阶段白屏。
- 浏览器构建产物只允许包含前端可运行模块，不应解析任何 Prisma Client 浏览器桩或服务端专用依赖。
- 正式部署环境中的首页访问不应被账单、退租等非当前页面的服务端领域模块连带阻断。

## 6. 实际结果
- 访问 `https://rento2026.top/` 后页面白屏，前端应用未正常渲染。
- 浏览器控制台报错：
  - `TypeError: Failed to resolve module specifier ".prisma/client/index-browser". Relative references must start with either "/", "./", or "../".`
- 报错发生在浏览器运行时，而不是服务端进程启动时报错，说明问题落点在前端首包或浏览器模块解析链路。

## 7. 影响范围初判
- 是否影响主链：是
- 是否影响历史数据：否
- 是否影响其他入口：是

## 8. 初步观察
- 当前现象更接近“前端浏览器包误引入了 Prisma 相关模块”，而不是单纯的 Caddy 配置、systemd 环境变量或服务端健康检查失败。
- 问题已经不是单页面局部错误，而是首页首屏白屏，说明前端入口或路由装载链路存在系统性边界污染。
- 初步证据显示，账务兼容包装层、退租结算共享逻辑与 Prisma 服务端领域模块之间可能存在前后端职责混写。
- 仅记录现象、约束和怀疑点，不在本文件中写实施方案。
