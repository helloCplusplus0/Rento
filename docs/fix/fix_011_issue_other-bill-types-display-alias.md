## 2. 背景
- 业务背景：
当前账单类型基本由：租金，押金，其他等组成，租金，押金类型是比较明确的，其他类型作为适应千差万别实际场景兜底的类型，这样的设计我觉得是正确的，但是当前存在一个问题还需要解决：其他类型的账单展示端没有提供别名入口，导致前端展示其他账单就直接展示其他，这样会给用户带来一定的困扰。比如：新增合同时根据合同信息会生成关联的其他类型账单，实际账单条目应该是；钥匙押金，卫生费，但是同一展示成其他，这样隐去了具体收费条目项，让用户困惑到底这是什么费。
