# Fix 013 Issue - PC 与移动端功能入口一致性差异

## 1. 问题摘要
- 问题编号：`fix_013`
- 发现阶段：真实场景验证
- 问题级别：`P1`
- 发现日期：2026-05-30
- 发现人：honlink

## 2. 背景
- 业务背景：
  - `fix_012` 已对移动端 UI 视觉进行了多轮分批收口，当前状态应视为阶段性完结，后续仍可能在新的轮次中继续重启移动端优化。
  - 在 `fix_012` 的执行过程中，部分页面为提升移动端展示效果，移除了低价值按钮入口、说明文本或压缩了局部结构。
  - 当前最优先的问题已不再是“某一个页面的字号或边距”，而是需要系统核对 PC 与移动端在正式业务功能入口上的一致性，避免出现“仅视觉优化，实际却改变了可达功能集合”的隐性分叉。
- 涉及模块：
  - 全局导航
  - 工作台快捷入口
  - 搜索与通知入口
  - 批量抄表页
  - 历史移动端专用组件治理
- 涉及页面 / API：
  - `/`
  - `/bills`
  - `/settings`
  - `/notifications`
  - `/meter-readings/batch`
- 涉及样例数据：
  - 当前真实场景验证环境中的工作台、导航与批量抄表页面数据

## 3. 复现前提
- 环境：本地开发环境 / 私有部署验证环境
- 账号：已登录后台的管理员账号
- 前置数据：
  - 至少存在账单、合同、房源、仪表与抄表数据
  - 工作台快捷入口与提醒卡已可正常渲染
- 是否稳定复现：是

## 4. 复现步骤
1. 分别在 PC 端与移动端打开工作台 `/`，对比全局导航、快捷入口、搜索、通知、设置、账单等入口是否同级可达。
2. 分别在 PC 端与移动端进入批量抄表页 `/meter-readings/batch`，对比同一仪表记录的合同、租客、警告和异常信息是否一致。
3. 检查代码实现中是否仍保留未实际接入的移动端专用入口组件或桌面端占位按钮。

## 5. 预期结果
- PC 与移动端只允许展示形态差异，不允许正式业务功能入口集合不一致。
- 同一正式能力在两端都应有可解释、可达、可执行的入口，不允许一端缺失、另一端补偿式暴露。
- 不允许继续保留未落地的占位按钮、废弃组件或仅一端存在的伪入口。

## 6. 实际结果
- 移动端底部导航当前展示 `设置`，而未展示 `账单`；桌面端主导航展示 `账单`，设置则通过右上角按钮进入。
- 桌面端右上角存在搜索与通知按钮，但当前仍是占位表达；移动端没有同级对应入口。
- 工作台快捷入口当前承担了部分移动端主导航缺失能力的补偿作用，例如 `账单管理` 仅通过工作台宫格直达。
- 批量抄表页的移动端卡片展示了更完整的合同编号、租客姓名、warning 文案；桌面端表格仅保留简化状态与异常提示。
- 仓库内仍保留 `MobileSearchBar.tsx` 这类未被当前正式路径接入的移动端专用组件。

## 7. 影响范围初判
- 是否影响主链：是
- 是否影响历史数据：否
- 是否影响其他入口：是

## 8. 初步观察
- 当前项目的明确约束已经升级为：
  - PC 与移动端只允许展示效果差异化，禁止功能入口出现不一致性。
  - PC 与移动端必须尽量通用一套前端，禁止演化为多套前端。
- 当前差异并非单点样式问题，而是“全局导航 + 工作台补偿入口 + 个别业务页信息不对齐 + 废弃组件残留”的组合问题。
- `fix_012` 的真相源仍然是“移动端视觉优化”；当前发现的问题已经超出纯视觉边界，适合独立进入新的 `fix_013` 闭环。
- 仅记录现象、约束和影响，不在本文件中写实施方案。
