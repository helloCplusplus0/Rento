# project_skills.md

> 职责：Rento 项目专属业务知识、迁移经验与验收习惯。
> 本文件只承接“本项目为什么这样做”，不重复承载通用工作方法与阶段状态。

## 1. 合同锚点技能
- `Contract` 是租务事实主锚点，账单、续租、退租、抄表与房态变化都应优先围绕合同建模。
- 页面交互若与合同主链冲突，应优先修正表达，不轻易重写数据模型。
- 删除、编辑、续租、退租等动作必须首先判断对合同链和账务链的影响。

## 2. 多仪表与历史保留技能
- 当前主线必须继续支持“一个房间可绑定多个仪表”。
- `Meter` 是独立资产，解绑或停用不等于删除历史。
- 对账单、抄表、合同、仪表历史，默认优先保留事实记录。
- 若必须移除当前关联，应优先建模为解绑、停用、终止或归档，而不是物理删除。

## 3. 删除门禁与账务语义技能
- 房间处于 `OCCUPIED` 或 `OVERDUE` 时，不应按普通空房直接删除。
- 合同未结束、账单未结清、仪表仍有关联历史时，删除操作必须被服务端门禁拦截。
- 必须区分 `monthlyRent`、`totalRent`、押金、首期应收、账单应收额等不同语义。
- 多仪表账单聚合逻辑必须可解释、可追溯、可拆回明细。

## 4. UI 承接技能
- 当前 `Rento` UI 展示效果是默认原型，迁移时优先承接页面信息结构、组件表达、交互节奏与整体风格。
- 非必要不改变展示效果；允许的调整只限宿主适配、明显 bug 修复、移动端可用性改善与最小信息架构优化。
- 若新页面仍包含说明卡片、开发态占位、宿主标签、重复入口或明显偏离旧页面结构，应视为 UI 漂移而不是“已完成迁移”。

## 5. 新旧宿主承接技能
- 旧 `Rento` 源码仍是当前仓库的直接参考基线，不需要复制第二份嵌入式源码区。
- 涉及页面迁移时，优先拆宿主协议、数据加载边界和运行时耦合，而不是重写页面主体表达。
- 涉及 API 或运行时迁移时，优先拆业务语义与宿主承载方式，再决定复用、适配或重写。

## 6. 数据与部署基线技能
- 当前正式数据访问主线是 `Prisma + PostgreSQL`；Prisma 替换不属于默认议题。
- 当前正式部署主线是 `Caddy + systemd + Hono + PostgreSQL + GitHub Release deploy bundle`。
- 云端默认不做源码构建，只负责下载、校验、解压、切换 `current`、刷新 `systemd/Caddy` 与健康检查。

## 7. PWA 承接技能
- PWA 目标不是做更激进的离线能力，而是把安装、更新、最小离线兜底与发布口径稳定迁到纯新主线。
- PWA 必须继续坚持“不缓存动态鉴权业务接口”的边界。
- 对 PWA 的正式判断，以带公认 HTTPS 证书的部署环境结果为准，而不是本地 HTTP 环境。

## 8. legacy 参考资产技能
- 旧容器化运行线与相关资产继续保留，但只承担历史实现参考、差异对照与未来归档前审计职责。
- 旧 `Rento` 当前价值是参考，而不是正式回退主路径。
- 是否归档或彻底移除 legacy 资产，应由后续独立治理任务决定，而不是在普通 fix 中顺手处理。

## 9. 项目验收技能
- 页面验收默认直接比较旧 `src/app/**/page.tsx` 与新宿主页面表达，而不是只看 URL 是否切换成功。
- API/query 验收默认以正式宿主是否已承担主职责、旧入口是否只剩兼容/参考职责为判断标准。
- cutover 验收默认要求正式入口、健康检查、主链 smoke、PWA 结果与 legacy 资产边界具备单一解释。
